Source code for ninjachef.recipes.sandbox

"""A recipe for development within a sandbox.

This setup is especially convenient for concurrent development of inter-related
and inter-dependent software projects. E.g. assume the following structure where
``foo`` and ``bar`` are projects, and ``bar`` is an executable that depends on
library ``foo``::

    root
    ├── sandbox
    ├── foo
    │   ├── include
    │   │   ├── foo.hpp
    │   │   └── …
    │   ├── source
    │   │   ├── foo.cpp
    │   │   └── …
    │   └── test
    │       └── …
    └── bar
        ├── source
        │   ├── main.cpp
        │   └── …
        └── test
            └── …

This can be put to use via the following::

    from pathlib import PurePath
    from ninjachef import Project, recipes, flavors

    root = PurePath('path', 'to', 'root')
    recipe = recipes.Sandbox(root, root / 'sandbox')
    foo = Project.read(root / 'foo')
    bar = Project.read(root / 'bar')

    ninja = recipe.cook([foo, bar], [flavors.Gcc()])
    print(ninja, end='')

All the build artifacts will appear inside ``sandbox``, with the following
structure (for a given `proj`):

    - headers are placed into ``sandbox/include/proj``

For each `flavor`:

    - library binaries are placed into ``sandbox/flavor/lib``
    - executable binaries are placed into ``sandbox/flavor/bin/proj/``
    - intermediate binaries are stowed inside ``sandbox/flavor/build/proj``

Note that it is not necessary for ``sandbox`` to live within ``root``, it can be
anywhere.

For our above example, this would result in the following sandbox structure::

    root
    ├── sandbox
    │   ├── include
    │   │   └── foo
    │   │       ├── foo.hpp
    │   │       └── …
    │   └── gcc
    │       ├── bin
    │       │   ├── foo
    │       │   │   └── test
    │       │   └── bar
    │       │       ├── bar
    │       │       └── test
    │       ├── lib
    │       │   └── libfoo.so
    │       └── build
    │           ├── foo
    │           │   ├── source
    │           │   │   ├── foo.o
    │           │   │   └── …
    │           │   └── test
    │           │       └── …
    │           └── bar
    │               └── source
    │                   ├── main.o
    │                   └── …
    ├── foo
    │   └── …
    └── bar
        └── …

The benefits of using a sandbox recipe are as follows:

    - the sand-boxed projects are completely left alone
    - it’s possible to work simultaneously on ``foo`` and ``bar`` even though
      the latter depends on the former without having to install ``foo`` or
      modify the system in any way
    - any change to e.g. a specific header of ``foo`` will only trigger the
      recompilation of those object files that depend on it inside both ``foo``
      and ``bar``.
    - any number of flavors (debug with compiler A, release with compiler A,
      debug with compiler B, etc.) can live side-by-side

There are also some drawbacks or caveats:

    - it is not a goal of this setup to tightly or perfectly isolate the
      projects from the rest of the environment/system; see how dependency
      resolution is handled
    - everything must be run from the root, it is not possible to descend into
      one of the sand-boxed projects and invoke Ninja there — there will be no
      Ninja build files inside
    - mixing flavors can require some work, e.g. if one of the sand-boxed
      projects require a different compiler than the rest

Recipe behavior:

    - The root of a project is assumed to be the name of the project (e.g. a
      project at ``path/to/foo`` will be named `foo`). This affects the `self`
      component name (see the :mod:`.cookbook`).
    - :class:`Dependencies <.Deps>` are first looked up amongst the sand-boxed
      projects. For a `Tuple` dependency the first element is assumed to be the
      project name, while the second element is assumed to be a component
      within that project; other elements, if any, are ignored. For this purpose
      :class:`str` dependencies (e.g. ``'foo'``) will be treated as if they
      were a pair of that :class:`str` (e.g. ``('foo', 'foo')``).

      If a project is found, but it has no appropriately named component,
      dependency resolution fails.

      If no appropriately named project is found, the dependency is assumed to
      be fulfilled out-of-sandbox.

      As a consequence, if a sand-boxed project shares a name with a particular
      library available on the system, it may easily end up shadowing it (for
      better or for worse).
"""

import io
from copy import copy
from itertools import chain
from collections import Counter
from pathlib import Path

from .recipe import Recipe, CyclicDependencyError, DuplicateNameError
from ..ninja_syntax import Writer as Ninja
from .. import Ingredient, Binary

def _component_name(proj, comp_name):
    if comp_name == 'self':
        return proj.name
    else:
        return comp_name

def _norm_dep(dep):
    if isinstance(dep, str):
        return (dep, dep)
    else:
        return dep

# We sort for the sake of humans, Ninja doesn't need it
def _toposorted(comps):
    sandbox = {_norm_dep(comp.name) for comp in comps}
    def any_in_sandbox(deps):
        return bool(deps & sandbox)

    graph = {comp: set(map(_norm_dep, comp.deps)) for comp in comps}
    workset = {comp for comp, deps in graph.items() if not any_in_sandbox(deps)}
    while workset:
        yield workset
        # Update graph to remove yielded entries and to remove yielded entries from pending dependencies
        graph = {
            comp: deps - set(map(lambda comp: (comp.proj.name, comp.name), workset))
            for comp, deps in graph.items()
            if comp not in workset
        }
        workset = {comp for comp, deps in graph.items() if not any_in_sandbox(deps)}

    if graph:
        raise CyclicDependencyError({(comp.proj.name, comp.name): deps for comp, deps in graph.items()})

class _Component(object):
    def __init__(self, proj, name, ingr):
        self.proj = proj
        self.name = _component_name(proj, name)
        self.ingr = ingr

    @property
    def deps(self):
        proj, ingr = self.proj, self.ingr

        deps = set(chain(*map(lambda d: d.project_deps.deps, proj.deps.values())))
        if isinstance(ingr, Binary):
            deps.update(ingr.deps.deps)
            descr = dict(filter(lambda comp: comp[1] is not ingr, proj.descr))
            rev_descr = {v: k for k, v in descr.items()}
            deps.update((proj.name, _component_name(proj, rev_descr[dep])) for dep in ingr.deps.local_deps)

        return deps


    def __repr__(self):
        message = "{cls}(proj=@{proj}, name=@{self.name})"
        return message.format(cls=type(self).__name__, self=self, proj=self.proj.name)


[docs]class Sandbox(Recipe):
    """Place build artifacts inside a common sandbox.

    Attributes:
        root (Path)
        sandbox (Path)

    Args:
        root (Union[str, Path]): Root path common to all projects that will be
            cooked. It won’t be possible to cook a project that lives outside
            this common prefix. This is also where the Ninja build file will be
            emitted. Can be absolute or relative.
        sandbox (Union[str, Path]): Path to the sandbox where headers and
            binaries will be placed. It can be entirely independent from `root`,
            and can be absolute or relative.
    """

    class _Env(object):
        def __init__(self, projs):
            def copy_proj(proj):
                proj = copy(proj)
                proj.name = Path(proj.path).resolve().name
                return proj

            projs = list(map(copy_proj, projs))

            comps = {
                _Component(proj, name, ingr)
                for proj in projs
                for name, ingr in chain(proj.headers.items(), proj.binaries.items())
            }

            # We index by project and name to make it easy to lookup sandbox-local dependencies.
            directory = {(comp.proj.name, comp.name): comp for comp in comps}
            reverse_directory = {comp.ingr: k for k, comp in directory.items()}
            assert len(directory) == len(reverse_directory), "ingredient mapping is not one-to-one"

            comps = list(chain(*_toposorted(comps)))

            self.deps, self.headers, self.binaries = Ingredient.split(comps, by='ingr')
            self.directory                         = directory
            self.reverse_directory                 = reverse_directory

        @property
        def comps(self):
            return list(chain(self.deps, self.headers, self.binaries))

        def __repr__(self):
            message = "{cls}(@comps=@{self.comps}, directory={self.directory}, reverse_directory={self.reverse_directory})"
            return message.format(cls=type(self).__name__, self=self)

    def __init__(self, root, sandbox):
        self.root     = Path(root)
        self.sandbox  = Path(sandbox)

    def __repr__(self):
        return "{cls}(root={self.root!r}, sandbox={self.sandbox!r})".format(cls=type(self).__name__, self=self)

    def _cook_headers(self, ninja, env, comp):
        proj    = comp.proj
        headers = comp.ingr

        absolute_proj = Path(proj.path).resolve()
        rebased_proj  = absolute_proj.relative_to(self.root.resolve())

        def prepare_header(path):
            """Express path relative to one of its roots, then graft onto sandbox path."""
            relative_path = path
            for prefix in headers.roots:
                try:
                    relative_path = relative_path.relative_to(prefix)
                    break
                except ValueError:
                    continue
            return (rebased_proj / path, self.sandbox / 'include' / relative_path)

        headers = set(map(prepare_header, headers.files.paths))

        for source, target in sorted(headers):
            ninja.build(str(target), self.rules.copy, inputs=str(source))

        # TODO this likely doesn't work for multiple Headers components
        name = proj.name + '-headers'
        ninja.build(name, self.rules.phony, inputs=sorted(map(lambda x: str(x[1]), headers)))
        return { name }

    def _cook_binary(self, ninja, env, flavor, comp):
        proj   = comp.proj
        name   = comp.name
        binary = comp.ingr

        component = (proj.name, name)
        absolute_proj = Path(proj.path).resolve()
        rebased_proj = absolute_proj.relative_to(self.root.resolve())

        def object_file(path):
            return (rebased_proj / path, self.sandbox / flavor.name / 'build' / proj.name / flavor.object_file(path))

        sources = set(map(object_file, chain(binary.files.paths, [binary.main]) if binary.main else binary.files.paths))

        variables = {
            'defines': binary.defines,
            # Make sure to put the project include paths before the sandbox include path
            'includes': sorted(map(lambda i: rebased_proj / i, binary.includes)) + [self.sandbox / 'include'],
            'cflags': [],
            'lflags': [],
            'lib_paths': [self.sandbox / flavor.name / 'lib'],
            'libs': [],
        }

        for source, path in sorted(sources):
            ninja.build(str(path), self.rule('compile', flavor), inputs=str(source),
                        variables=flavor.variables(variables, component=component))

        def exe(name, proj=proj.name):
            return str(self.sandbox / flavor.name / 'bin' / proj / flavor.exe_name(name))

        def lib(name):
            return str(self.sandbox / flavor.name / 'lib' / flavor.lib_name(name))

        def target(binary, name, proj=proj.name):
            if binary.main:
                return exe(name, proj=proj)
            else:
                return lib(name)

        link = self.rule('link_exe', flavor) if binary.main else self.rule('link_lib', flavor)

        variables['libs'] = libs = []
        sandbox_libs = []
        for dep in chain(*(list(map(lambda deps: deps.project_deps.deps, proj.deps.values())) + [binary.deps.deps])):
            dep = _norm_dep(dep)
            if dep in env.directory:
                comp = env.directory[dep]
                assert isinstance(comp.ingr, Binary), "cannot handle non-binary dependency"
                libs.append(comp.name)
                sandbox_libs.append(target(comp.ingr, comp.name, comp.proj.name))
            else:
                libs.append(dep[0])
        for dep in binary.deps.local_deps:
            libs.append(env.reverse_directory[dep][1])
            sandbox_libs.append(target(dep, env.reverse_directory[dep][1]))

        tgt = target(binary, name)
        ninja.build(tgt, link, inputs=sorted(map(lambda x: str(x[1]), sources)),
                    implicit=sandbox_libs,
                    variables=flavor.variables(variables, component=component))
        return { flavor.name: [tgt] }

    def _cook_aliases(self, ninja, env, flavors, headers, binaries):
        for flavor in flavors:
            ninja.build(flavor.name, self.rules.phony, list(chain(headers, binaries[flavor.name])))
            ninja.newline()


[docs]    def rule(self, name, flavor):
        if name not in self.rules:
            raise ValueError("Rule {name!r} is not supported".format(name=name))
        return flavor.name + '-' + name



[docs]    def cook(self, projs, flavors, comments=True, default_targets=None):
        out = io.StringIO()
        ninja = Ninja(out)
        env = self._Env(projs)
        flavors = list(flavors)
        names = list(map(lambda f: f.name, flavors))
        if len(names) != len(set(names)):
            duplicates = [n for n, c in Counter(names).items() if c > 1]
            raise DuplicateNameError(duplicates)

        def section(title, level=1):
            if comments:
                guard = '#' * (4 - level)
                ninja.output.write("{guard} {title}\n".format(**locals()))

        def merge_update(dest, source):
            dest.update(chain(source.items(),
                              ((k, dest[k] + source[k]) for k in source.keys() & dest.keys())))

        section('Rules')
        ninja.rule(self.rules.copy_to,
                   command='cp --parents $in $destination',
                   description='Copying to $destination')
        ninja.rule(self.rules.copy,
                   command='cp $in $out',
                   description='Copying $in')
        for flavor in flavors:
            section(flavor.name, level=2)
            flavor.ninja_rules(ninja, self)
        ninja.newline()

        header_targets = set()
        section('Headers')
        for header in env.headers:
            header_targets.update(self._cook_headers(ninja, env, header))
            ninja.newline()

        binary_targets = dict()
        section('Binaries')
        for flavor in flavors:
            section(flavor.name, level=2)
            for binary in env.binaries:
                merge_update(binary_targets, self._cook_binary(ninja, env, flavor, binary))
                ninja.newline()

        section('Aliases')
        self._cook_aliases(ninja, env, flavors, header_targets, binary_targets)

        if default_targets:
            ninja.default(default_targets)

        return out.getvalue()
