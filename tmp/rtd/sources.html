Sources¶

Source range functions are provided to produce Range models. Typical source range functions create a given range from certain parameters (e.g. interval for half-open intervals) or can adapt pre-existing code. For instance container makes it possible to ‘turn’ a container such as std::vector<T> into a model of Range.

template<Variable Policy, ForwardableType Functor>
constexpr generate_range<Functor, Policy> operator()(Policy policy, Functor &&functor) const¶

Create a range over the Nullable values repeatedly produced by functor, up to the first null value (exclusive).

Important

The default_generate policy is safe by default, but disables copies. While Functor may be copyable, it is not possible to check without user intervention whether copying correctly preserves range state. The user can assert that Functor copies do provide this guarantee by specifying the relaxed_generate policy.

The following demonstrates the potential pitfall when proper care is not taken:

int shared_state = 0;
auto func = [&shared_state] { return shared_state < 6 ? make_optional(shared_state++) : nothing; };
auto r = generate(relaxed_generate, func);

// func is copyable, so r is as well
auto s = r;

// since all copies of func close over the one and only shared_state variable,
// consuming r wrongfully affects s at a distance!
for(auto ignore: in_place(r)) static_cast<void>(ignore);

// now s is empty even though it has not been ranged over yet
Parameters:
  • policy – a policy object, either default_generate or relaxed_generate
  • functor – a function object satisfying both Invokable<Functor&> and Nullable<result<Functor&>>

constexpr container_range<std::decay_t<Iter>> operator()(Iter &&iter) const¶

Warning

This library feature is much more experimental than the rest, and is considered incomplete or unsatisfactory. It is likely to be subject to significant changes or dropped altogether.

Create a range over the elements of iter.

Note

iter will be copied or moved and made part of the resulting range, to provide backing for the elements. If only a view into iter is needed, as_range should be used instead.

Warning

The resulting range is rooted and neither Copyable nor Movable. Its positions are only valid for the particular object they were created from.

As a possible alternative, indexable provides similar functionality albeit for containers which can be indexed (e.g. arrays), and is not rooted.

constexpr container_range<Iter> copy() const& requires CopyConstructible<Iter>:

Named copy constructor.

Create a fresh container_range<Iter> copy of *this, with its own backing container. Constant time complexity if Iter provides random-access iterators, linear otherwise.

Note

This functionality is not provided as a copy constructor since the fresh range must use its own positions. See Ranges and copying/moving.

constexpr container_range<Iter> move() & requires MoveConstructible<Iter>:

Named move constructor. A moving variant of copy().

Note

This functionality is not provided as a move constructor since the fresh range must use its own positions. See Ranges and copying/moving.

constexpr container_range_source<Iter> transfer() const& requires CopyConstructible<Iter>:

A variant of copy() that makes it possible to return a container_range<Iter>, as it is constructible from container_range_source<Iter>:

auto example(container_range<some_iterable> arg)
{
    // not possible as container_range<some_iterable> is not move constructible:
    //     return arg.move();
    // instead:
    return container_range { arg.transfer() };
}
constexpr container_range_source<Iter> transfer() && requires MoveConstructible<Iter>:
