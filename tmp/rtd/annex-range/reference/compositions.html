Compositions¶

A fixed and finite number of ranges can be composed together to form a new range. This is to be contrasted with Arrangements, where a variable and not necessarily finite number of ranges gives rise to a new range. Some forms of composition have an arrangement counterpart.

concat¶

Provided by annex/range/composition/concat.hpp.

Contexts{...Ctxs}
using concat_range = bounded_context<concat_context<Ctxs...>>¶
Additional requirements

There are common types for the element types, i.e. the following must be valid:

using peek_t    = common_type_t<peek_element_t<Ctxs>...>;
using element_t = common_type_t<element_t<Ctxs>...>;

peek_t must be convertible to element_t const&.

Successive composition of ranges. E.g. the result of concat(a, b, c) ranges over the elements of first a, then b, and finally c in order.

constexpr functors::zip zip¶

This constant is a Range function with one or more associated simple models:

Contexts{... Ctxs} zip_range

MoveConstructibles{...Rngs}
constexpr zip_range<context_t<Rngs>...> operator()(Rngs... rngs) const¶
Notation
using... Ctxs = context_t<Rngs>...;

auto&... ctxs = rngs.context...;

Side-by-side composition of ranges. E.g. the result of zip(a, b, c) first yields the tuple { a0, b0, c0 } made up of the respective first elements of a, b, and c; then the tuple { a1, b1, c1 } and so on.

The resulting range will be as long as the shortest argument range amongst rngs.
