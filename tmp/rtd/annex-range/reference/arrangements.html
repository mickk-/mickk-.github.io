Arrangements¶

A variable and not necessarily finite number of ranges, typically in the guise of a Traversable of ranges, can be arranged together to form a new range. This is to be contrasted with Compositions, where a fixed and finite number of ranges gives rise to a new range. Some arrangements have a composition counterpart.

constexpr functors::flatten flatten¶

This constant is a Range function with one or more associated simple models:

Context{Ctx} flatten_range

MoveConstructible{Rng}
constexpr flatten_range<context_t<Rng>> operator()(Rng rng) const¶
Notation
using OuterCtx = context_t<Rng>;
using InnerCtx = context_t<as_range_t<element_t<OuterCtx>>>;
auto&& ctx = rng.context;

Create a range over the inner elements of the outer sequence elements of rng.

The element_t<OuterCtx> elements may be models of Range, or anything that is a valid argument to as_range. This means that flatten(rng) behaves as if it performed a pre-processing step of sorts, resulting in a behaviour nearly equivalent to flat_map(as_range, rng) as a convenience to the user.

Note that because as_range only ever accepts lvalue Iterable arguments, it is not possible to flatten a range that produces Iterable objects. This is due to the Ranges and copying/moving rules (see the warning for container). Combining indexable with flatten is however safe and may be used:

auto r = flat_map(indexable, rng);

Since container does not even produce a Movable range, it cannot be used with flatten.
