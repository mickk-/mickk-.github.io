Consumers¶

There are various ways to make use of a Range. They are overall referred to as consumers and can be distinguished by the various situations in which ranges are meant to be used:

In range-based for statements¶

Provided by annex/range/range.hpp.

Range{Rng}
using loop_type¶

A special type to make a Range usable in a range-based for statement. Ranges are not directly usable that way due to limitations of the language.

Note

loop_type is for use in for statements only, it is not a model of Iterable. range_iterator_pair is such a type.

See also

loop, inplace_loop, as_iterators

constexpr functors::loop const &loop¶

This constant is a Function with one or more associated simple models:

Range{Rng} loop_type
MoveConstructible{Rng}
constexpr loop_type<Rng> operator()(Rng rng) const¶

Make rng available for use in a range-based for statement. This does not modify rng:

for(auto&& elem: range::loop(rng)) {
    // use each element of rng successively
}

for(auto&& elem: range::loop(rng)) {
    // use each element of rng successively, again
}

The result of loop will however be consumed in the operation:

auto lp = range::loop(rng);
for(auto&& elem: lp) {
    // use each element of rng succesively
}

// rng is untouched, but lp is done
for(auto&& elem: lp) {
    // not reached
    assert( false );
}
Simple model members:
 looped_range: rng

See also

inplace_loop:
In-place variant.
as_iterators:
For compatibility with iterator-based code.

For compatibility with iterator-taking containers.
constexpr functors::inplace_loop const &inplace_loop¶
Type{Rng}
constexpr loop_type<Rng&> operator()(Rng &rng) const¶

Make rng available for in-place use in a range-based for statement. ‘In-place’ means that rng will be mutated in the process. In particular a full, uninterrupted iteration through all the elements of rng means that equal_pos(rng.context, rng.from, rng.to) will hold after the for statement:

for(auto&& elem: range::inplace_loop(rng)) {
    // use each element of rng successively
}
// rng is now empty, the result of inplace_loop is done as well
Simple model members:
 looped_range: rng

Notation:

Collect the elements of rng into a container, which type will be a specialization of Cont. The elements will be decayed and stored as stored_t.

Compatibility with iterator code¶

Provided by annex/range/range.hpp.

ContextType{Ctx}
using range_iterator¶
Additional requirements:
 SameType<position_t<Ctx>, sentinel_t<Ctx>>

A ‘fat’ iterator adaptor bridging the gap between range-based code and iterator-based code. ‘Fat’ means that in addition to storing position-specific information the iterator also references the underlying Context of the range. As a consequence, that context must outlive any iterator that references it.

See also

as_iterators

constexpr functors::as_iterators const &as_iterators¶
This constant is a Function with one or more associated simple models:
ContextType{Ctx} range_iterator_pair
ContextType{Ctx}
constexpr range_iterator_pair<Ctx> operator()(Ctx &ctx, position_t<Ctx> from, sentinel_t<Ctx> to) const¶

Warning

This is a feature of last resort. It may be hard to use correctly or otherwise undesirable, and is only provided for situations where no alternative is possible.

In particular, ctx must outlive the result of as_iterators.

Create a model of Iterable over the range defined by \(\left\{ctx, from, to\right\}\). This makes it possible to use the range with iterator-based code:

auto iter = range::as_iterators(ctx, from, to);
// code that takes an Iterable
iterable_code(iter);
// code that takes a pair of Iterators explicitly
std::sort(iter.first, iter.last);

Due to the differences between range and iterator concepts, there are limitations on when as_iterators may be used.

Parameters:ctx – A range context with identical position and sentinel types.

Algorithms¶

Folds¶

Provided by annex/range/consumer/fold.hpp.

constexpr functors::fold const &fold¶

This constant is a Function.

template<typename Combiner, Movable Seed, Range Rng>
constexpr Seed operator()(Combiner &&combiner, Seed seed, Rng rng) const¶

Combines all the elements of rng according to combiner starting with the initial seed. Using e0, e1, ..., e_n to denote the elements of rng, this amounts to computing:

auto result = combiner(...combiner(combiner(std::move(seed), e0), e1)..., e_n);

(Except that invoke_as<Seed> is used for the purpose of invoking.)

In the case that rng is empty, the result is seed.

Parameters:
  • combiner – A function object satisfying InvokableAs<Seed, Combiner, Seed, element_t<context_t<Rng>>>. The parameter order is significant.
  • seed – The initial value to grow with combiner.
  • rng – A Range which elements will each be read once, in order.
