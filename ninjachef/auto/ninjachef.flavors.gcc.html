rules¶

Dict[Union[str, Tuple[str, ...]], Dict[str, Any]]

A mapping of project- or component-specific rules. Rules take the form of a { 'defines': defs, 'cflags': flags, 'lflags': flags, 'runflags': flags } dictionary, where defs is a set of str and flags are lists of str just like keyword arguments cflags and lflags. It is possible to leave out empty entries, i.e. {} form valid rules. See Testsuite for the meaning of runflags and how multiple runflags are merged together.

The keys in the mapping take the form of a str, when referring to a project, or of a tuple of str when referring to a more specific component. The tuple will be interpreted as it is for Deps.

When a component is prepared with this flavor, the rules will be looked up to see if there are additional defines and flags specified either for the project the component belongs to or or for the component itself. All applicable rules will be merged together. Example:

gcc = flavors.Gcc(…)
# Project-specific
gcc.rules['foo'] = {'defines': {'FOO'}}
# More finely-grained
gcc.rules[('foo', 'bar')] = {'defines': {'BAR'}}
# both FOO and BAR will be defined during compilation of (foo, bar)
resolver¶

Dict[str, Callable[[str], str]]

A mapping of glob-style str patterns (e.g. ‘boost_*’) to callbacks. This allows mapping the dependencies as they appear in the ingredient files to the actual system libraries. When a dependency matches a pattern of the resolver, the associated callback will be used to transform it.

Parameters:
  • name (Optional[str]) – The name of the flavor, to distinguish it from other flavors.
  • executable (Optional[str]) – The GCC executable to use.
  • cflags (Optional[Sequence[str]]) – The GCC flags to use during compilation and linking. (There is no way to specify a flag during compilation and not linking, but note that the GCC manual recommends against doing that anyway—redundant flags during linking are harmless.)
  • lflags (Optional[Sequence[str]]) – The GCC flags to use during linking, but not compilation.
